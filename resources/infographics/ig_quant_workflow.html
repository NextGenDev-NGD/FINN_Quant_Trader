FINN Quantitative Workflow

9-step process from hypothesis to live trade — repeatable, testable, documented
Tier 2 · FINN-Specific

FINN framing rule: Every trade starts as a hypothesis. A hypothesis is only as good as the test behind it. FINN's workflow exists so that the process from "I think X works" to "I am trading X" is never skipped, compressed, or guessed. No IC, no trade. No out-of-sample test, no capital allocation.

The 9-Step Workflow
Research
1
Define the Hypothesis
Python · Markdown
State the edge in one testable sentence: "When X condition is true, Y outcome occurs Z% of the time." If you can't write the null hypothesis, you don't have a hypothesis.
Name the signal Define universe Set lookback window Write null hypothesis
Output: Hypothesis statement + null in plain English
2
Pull and Clean the Data
yfinance · pandas
Download OHLCV data for the full lookback period. Flatten columns, handle splits/dividends, fill or drop NaNs. Log your data source and date range.
yf.download() Flatten columns Check for gaps Document date range
Output: Clean OHLCV DataFrame with verified index
◆ Gate: Data complete and gap-free? If not, fix before continuing.
Build
3
Engineer the Features
pandas_ta · pandas
Compute the technical indicators and derived features that define your signal. Each feature must have a clear role — no kitchen-sink feature dumps.
pandas_ta indicators Rolling windows Lag variables Target variable
Output: Feature DataFrame — each column named and documented
4
Run the In-Sample Backtest
xgboost · scikit-learn
Train on the in-sample period (first 70–80% of data). Generate signals. Compute IC, win rate, Sharpe ratio, and max drawdown. This is your baseline, not your truth.
Train/test split (time-based) IC calculation Signal generation Equity curve
Output: IS performance metrics — IC, Sharpe, drawdown, win rate
◆ Gate: IC > 0.05 in-sample? If not, revise hypothesis — do not proceed to OOS.
Test
5
Out-of-Sample Validation
scikit-learn · pandas
Evaluate the model on the held-out period (last 20–30% of data). No retraining allowed. If IC collapses OOS, the in-sample edge was overfitting, not a real edge.
Apply IS model to OOS Compute OOS IC Compare IS vs. OOS Sharpe Log decay rate
Output: OOS performance metrics — IC, Sharpe, drawdown vs. IS
6
Stress Test the Edge
pandas · numpy
Test the strategy across different market regimes (trending, choppy, crisis), vary key parameters ±20%, and confirm the edge isn't parameter-sensitive fragility.
Regime slicing Parameter sensitivity Drawdown stress test Transaction cost impact
Output: Stress test report — performance across regimes, parameter stability
◆ Gate: OOS IC > 0.03 AND stable across ±20% parameter variation? If not, back to Step 3.
7
Paper Trade the Signal
Webull · Signal log
Run the live signal in Webull paper trading for a minimum of 20 trades or 4 weeks, whichever is longer. Record every entry, exit, and reason in the signal log.
Live signal generation Paper orders Daily log entry Compare to backtest
Output: signal_report_YYYYMMDD.csv — live vs. backtest IC comparison
◆ Gate: Paper trade IC within 25% of backtest IC? Live drawdown acceptable? Proceed to capital allocation.
Trade
8
Size and Risk Allocation
Risk rules · Position sizing
Apply FINN's 6 risk rules to determine position size, max exposure, and stop placement before the first live trade. Capital allocation is a function of IC, not conviction.
Max 2% portfolio risk/trade ATR-based stops Sector concentration limit Kelly sizing (if IC validated)
Output: Position size per trade, max drawdown threshold for shutdown
9
Monitor, Log, and Iterate
all_trades.csv · monthly review
Log every live trade in all_trades_[initials].csv. Review IC monthly. If live IC decays below threshold for 2 consecutive months, stop trading and return to Step 5 (re-validate).
Daily P&L log Monthly IC review Sharpe trending? Iteration trigger
Output: Monthly performance report — live IC, Sharpe, drawdown vs. target
