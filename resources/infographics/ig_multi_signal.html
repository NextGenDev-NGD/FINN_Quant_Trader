Multi-Signal Scoring

Weighted composite score — combining signals, applying veto logic, and sizing to conviction
Tier 2 · FINN-Specific

FINN framing rule: A single signal with IC 0.05 is a guess with better-than-random odds. Three uncorrelated signals combined into a weighted composite can achieve an IC that none of them could alone. But signals must be combined deliberately — not stacked, not averaged blindly, and not allowed to vote if a veto condition is active. Build the composite. Enforce the vetoes. Size to the score.

Composite Score Formula
Score = Σ(signal_i × weight_i)
      subject to: veto conditions = False
Range: 0.0 – 1.0 (long) | −1.0 – 0.0 (short)
Signal Normalization Rules
T
Trend: normalize to [0, 1]
1 = strong uptrend, 0 = strong downtrend
Use EMA slope percentile rank over 252 days
M
Momentum: normalize to [0, 1]
RSI: (RSI − 30) / 40, clipped to [0,1]
MACD: sign + normalized histogram magnitude
V
Volume: normalize to [0, 1]
vol_ratio = current / 20d avg, percentile rank
0.5 = average, >0.7 = confirming volume

Extreme greed → 0.0 (bearish contrarian)

No pattern or bearish → 0.0

⊘ Veto Logic — Score Overrides

A veto condition forces the composite score to 0.5 (neutral/no trade) regardless of what the other signals say. Vetoes exist because certain conditions invalidate the signal entirely — not just reduce its confidence.

Trend Veto
Counter-trend signal
If daily trend is down but composite score > 0.60, Trend Veto overrides. Score → 0.5. Reason: you cannot go long against a confirmed daily downtrend regardless of sub-signal alignment.
Volume Veto
Volume ratio < 0.5 on breakout
If a breakout signal fires but 20-day volume ratio is below 0.5 (below-average participation), Volume Veto overrides. Score → 0.5. Low-volume breakouts fail at an unacceptable rate.
Earnings Veto
Earnings within 3 trading days
If the target stock has an earnings event within 3 trading days, all long/short signals are vetoed. Earnings binary risk invalidates any pre-earnings technical signal.
Python Reference — Composite Score Calculation
multi_signal_composite.py — FINN scoring engine
Python 3.11
import pandas as pd
import numpy as np

WEIGHTS = {
    'trend':     0.30,
    'momentum':  0.25,
    'volume':    0.25,
    'sentiment': 0.10,
    'pattern':   0.10,
}

def composite_score(signals: dict, vetoes: dict) -> float:
    # Enforce vetoes first
    if vetoes.get('trend') or vetoes.get('volume') or vetoes.get('earnings'):
        return 0.5  # neutral — no trade

    score = sum(signals[k] * WEIGHTS[k] for k in WEIGHTS if k in signals)
    return np.clip(score, 0.0, 1.0)

def position_size_multiplier(score: float) -> float:
    if   score >= 0.80: return 1.00
    elif score >= 0.60: return 0.75
    elif score >= 0.45: return 0.50
    else:               return 0.00  # no trade

# Correlation guard — run before combining signals
def check_signal_correlation(signal_df: pd.DataFrame, threshold=0.6) -> bool:
    corr = signal_df.corr()
    mask = np.triu(np.ones(corr.shape, dtype=bool), k=1)
    return (corr.where(mask).abs() < threshold).all().all()
